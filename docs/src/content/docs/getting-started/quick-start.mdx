One command from zero to a running team in your Discord server.

:::danger[Security — read before running]
By default, disclaw-team runs Claude Code with `--dangerously-skip-permissions`. **Anyone who can send messages in your Discord server can trigger commands on your machine** — file reads, writes, shell execution, and more. Only run on servers where you trust every member. Use `--safe` mode for approval-based operation, or run on an isolated machine. See [Safe Mode](../concepts/safe-mode) for details.
:::

## Run the setup

```bash
npx disclaw-team init
```

This installs disclaw-team, runs the setup wizard, and launches your bots. Have your Discord bot tokens ready from [Discord Setup](./discord-setup) before running.

:::note[First run restart]
On first launch, bots restart once automatically to pick up Discord user IDs. This is expected — see [Your First Team](./your-first-team) for details.
:::

## Open the dashboard

Once init completes, open [http://localhost:5173](http://localhost:5173). The onboarding wizard walks you through:

- Discord server and channel configuration
- Bot token setup
- Role assignment (pick from 22 preloaded roles)
- Team preset selection

After onboarding, the dashboard is the primary way to manage your team day-to-day:

- Assign and swap roles visually
- Start/stop/restart individual bots or the whole team
- Switch between team presets with one click
- Create and edit custom roles
- Set up recurring schedules
- Monitor live status

See [Using the Web Dashboard](../guides/web-dashboard) for a full walkthrough.

## You're live

Your bots are running in Discord. Try @mentioning the orchestrator bot to kick off a task.
